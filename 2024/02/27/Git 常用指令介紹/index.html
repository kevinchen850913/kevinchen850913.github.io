Git 常用指令介紹

發表於2024-02-27|更新於2024-11-14

# 事前準備工作

# terminal 指令表

Windows MacOS/Linux 功能
cd cd 切換目錄
cd pwd 顯示目前所在資料夾
dir ls 目前資料夾的檔案列表
md mkdir 創建資料夾
type nul> touch 建立檔案
copy cp 複製檔案
move mv 移動檔案
del rm 刪除檔案
cls clear 清除 Terminal 內容

# git 常用指令表

git cmd 功能
git clone • Copy from REMOTE repo
git branch • Review how many branches have • Create NEW branch from current/specific branch
git status • Check the status of your LOCAL GIT repository
git pull • Bring LOCAL branch up-to-date
git checkout • Switch branch • Create NEW branch from current/specific branch
git add • Put all the new file you just add and file you modified to the LOCAL stage area.
git commit • Commit your code to the LOCAL repository
git push • Check the status of your LOCAL GIT repository
git log • Check the status of your LOCAL GIT repository
git rebase • Reapply commits on top of another base tip

# git 下載與安裝

安裝 Git 請到官方網站下載合適的版本。
Git - Downloads (git-scm.com)

安裝完後驗證 Git 是否安裝成功，以及確認版本資訊對不對。

# config 使用者設定

如果你是第一次使用 Git ，需要設定使用者的 Email 信箱以及使用者名稱。

輸入完成之後，可以檢視目前的設定是否正確。

因為我有安裝 Git 相關的圖形化介面工具，所以 git config --list 這個指令還會輸出其它的設定，下面是可能會看到的畫面。

# 本地相關指令

# Init 初始化

確認設定完成後，在目標資料夾中執行下方的指令，就可以資料夾變成 git 庫。

輸入該指令後，資料夾中會多出 .git 的隱藏檔案，版本控管文件都存於此資料夾內，如果刪除此資料夾，就無法復原版本控管的檔案。

# Status 狀態

確認目前資料夾狀態，在資料夾中建立 myfile.txt 再確認工作區的狀態。

回傳訊息會告訴你，當前存在未在索引中的檔案。

# Add 加入

此命令可將指定的檔案從工作區加入暫存區。

測試中將 myfile.txt 從工作區加入到的暫存區，再使用 Status 就可以看到 myfile.txt 已經被加入暫存區。

如果不想將檔案加入暫存區，可以用 clean -f 指令把沒有版控的檔案刪除。

# Commit 提交

Commit 指令可以將暫存區的檔案移動至儲存區，儲存區的檔案才會被列版本控管，有版本控管才有辦法回朔至該狀態，所以記得要常常進行提交，以免資料遺失。

在 git 的規則中，檔案可以未有任何改動提交一定要有提交說明，而 -m 的意思是後方的文字是 Commit 中的訊息，如果沒有使用 -m 還是會被要求要寫說明。

指令完成後確認狀態，訊息顯示當前當前狀態與上次提交並無不同。

如果 commit 的訊息有錯誤，將指令加上 –amend 可以修改最後一次的訊息。

前一次提交為空白提交，所以出現下方訊息，系統建議退回其一個 Commit。

將檔案修改後再下達指令，即可成功覆寫過去錯誤的訊息。

反之若有檔案未加入 Commit ，–amend –no-edit 可以將暫存區的檔案併入前一筆。

# Log 歷史紀錄

Log 功能可以查詢所有 commit 紀錄。

加上參數 -p 顯示 commit 詳細資訊。

加上參數 –oneline 顯示更精簡的 commit 紀錄。

加上參數 –graph 顯示 commit 線圖。

同時加上 –oneline 與 –graph 顯示精簡的線圖。

加上參數 –all 顯示包含 branch 的紀錄 (目前沒有分支)

# 分支相關指令

# Branch 操作分支

branch 指令可以查看所有分支。

branch 後方加上分支名稱，可以新增分支。

加上 -d 可以刪除分支。

# Checkout 切換分支

checkout 加上分支名稱，可以將 HEAD 移動到該分支。

checkout 加上 -b 與分支名稱，可以新增分支並同時移動 HEAD。

# Example - switch branch

# 遠端相關指令

# Remote 遠端庫

首先要設定遠端庫的資訊，幫該遠端命名與提供路徑。

設定為遠端庫，設定成功後查看遠端庫資訊。

先將本地資料上推後，再查看遠端分支。

# Clone 副本

副本遠端數據庫，先將遠端的資料拉到本地，再開始修改本地的檔案。

測試將 openbmc 副本到本地的資料夾。

# Pull 拉取

  • fetch 從遠端數據庫下載 commit 紀錄。
  • pull = fetch + merge，從遠端數據庫下載 commit
    紀錄並合併至本地資料。

因為遠端沒有被更動，所以什麼事都沒發生。

遠端被更新後可以看到本地下載了新的資料。

# Push 推送

將本地的提交上傳至遠端，要注意是否會與遠端提交產生衝突。

提交一些改動，然後 Push 至遠端。

如果遠端產生衝突，會出現類似下方的訊息。

強制將衝突分支推至遠端，加上 -f 可以強制推送，不過不建議這樣處理衝突。

# Example - clone openbmc

# 合併與基變指令

# Merge 合併

使用 merge 將 develop 合併到目前分支 (master)。

使用 merge 將 develop 合併到目前分支 (master)，但不使用 –no-ff 快進合併，此時會多出 Merge 專屬的 Commit ，此 Commit 是系統自動產生。

# Rebase 基變

假設我們要將 develop 分支 Rebase 到 mester，則先切換到 develop，然後輸入指令。

要合併用 Rebase 盡量少用 Merge，Rebase 有著非常多取巧的用法，但若是不使用 GUI 工具，我完全不會想用 Terminal 來變魔術，一定需要搭配 GUI 工具，比較不容易玩壞。

Rebase 的取巧的用法

  • Rebase 可以對同 Branch 中的 Commit 執行。
  • Rebase 可以分割 Commit，使其改動範圍更加聚焦。
  • 程式碼改壞了，可以對其 Rebase 移除錯誤修改。
  • 延續性的問題可以用 Rebase 合併至同一個 Commit。
  • 可以用 Rebase 使某些 Commit 帶有特殊功能 (客製版)。
