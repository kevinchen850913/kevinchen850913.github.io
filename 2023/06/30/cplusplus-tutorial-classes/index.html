發表於2023-06-30|更新於2024-11-14

# 類別

類與數據結構一樣可以包含數據成員，但它還可以包含函數作為成員。

具有相同的格式，除了它們還可以包含函數並具有這些稱為訪問說明符的新事物。
訪問說明符是以下三個關鍵字之一 private、public、protected

  • private : 類的成員只能從同一類的其他成員訪問。
  • protected : 成員可以從同一類的其他成員訪問，也可以從他們的派生類的成員訪問。
  • public : 可以從對象可見的任何地方訪問成員。

默認情況下，使用關鍵字聲明的類的所有成員 class 對其所有成員都具有私有訪問權限。

只需在對象名稱和成員名稱之間插入一個點，就可以像訪問普通函數或普通變量一樣使用。

rect 不能從類外部訪問的成員是 width 、  height ，因為它們具有私有訪問權限，並且只能從同一類的其他成員內部引用它們。

成員函數的定義 area 已直接包含在類的定義中。 Rectangle 相反， set_values 它只是在類中用它的原型聲明，但它的定義在類之外。在此外部定義中，範圍運算符 (  :: ) 用於指定正在定義的函數是類的成員 Rectangle ，而不是常規的非成員函數。

類最重要的屬性是它是一種類型，因此我們可以聲明它的多個對象。例如，在前面的 class 示例之後，我們可以在 object 之外 Rectangle 聲明 object ： rectb``rect

類允許使用面向對象的範例進行編程：數據和函數都是對象的成員，減少了傳遞和攜帶處理程序或其他狀態變量作為函數參數的需要，因為它們是其成員被調用的對象的一部分。

在調用 rect.areaor 時沒有傳遞任何參數。這些成員函數直接使用各自對象的數據成員 rect 和 rectb.

# 構造器

一個類可以包含一個稱為其構造函數的特殊函數，每當創建該類的新對象時都會自動調用該函數，從而允許該類初始化成員變量或分配存儲空間。這個構造函數的聲明就像一個普通的成員函數，但是有一個與類名匹配的名字並且沒有任何返回類型。

構造函數不能像普通成員函數一樣被顯式調用。當創建該類的新對象時，它們只執行一次。

# 統一初始化

C++ 引入了使用統一初始化調用構造函數的可能性，它本質上與函數形式相同，但使用大括號 {} 而不是圓括號 ()。

統一初始化優於函數形式的一個優點是，與圓括號不同，大括號不能與函數聲明混淆，因此可用於顯式調用默認構造函數。

選擇調用構造函數的語法主要是風格問題。大多數現有代碼目前使用函數形式，一些較新的樣式指南建議選擇統一初始化而不是其他代碼。
