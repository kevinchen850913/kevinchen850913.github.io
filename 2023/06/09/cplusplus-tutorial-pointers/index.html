C++ 學習筆記 - 指標(pointers)

發表於2023-06-09|更新於2023-10-29
|閱讀量:

# 指標

C++ 程式不會主動決定變量的內存地址，這個任務通常由運行時的操作系統來執行。然而，我們可以通過取址運算符獲取變量的地址，並使用指針來訪問和修改該地址上的數據。通過這種方式，我們可以在程序運行時動態地操作內存中的特定位置，實現更靈活和動態的代碼行為。

# 地址 & ，指向值 *

  • 變量的地址可以通過使用取址運算符 & 來獲取。
  • 接訪問它們指向的變量透過取消引用運算符 * 來完成。
  • * 運算符本身可以理解為指向的值。

在下方的範例中，myvar 的地址為 1776。

關於 * 的位置我個人習慣是貼近類型優先，因為比較契合中文的邏輯， ptr 的型別是 int* ，而不是 ptr 指向的型別是 int ，不過程式中三種寫法都能被接受。

建議單獨宣告指標，避免不必要的麻煩。

# 指標應用範例

  • firstvalue 與 secondvalue 在程式中沒有直接設置任何值。
  • 使用解引用運算符 * 的表達式的含義與不使用解引用運算符的含義非常不同。

# 指標和數組

mypointer 和 myarray 有非常相似的屬性。

  • mypointer 可以分配不同的地址，
  • myarray 不能分配地址，只能代表相同的 20 個元素。

括號 ( [] ) 被解釋為指定數組元素的索引。
實際上括號是一個取消引用運算符，稱為偏移運算符。
括號類似於 * ，但還將括號之間的數字添加到地址。

這兩個表達式是等價且有效的，一個數組的名稱可以像指向其第一個元素的指針一樣使用。

# 指標運算

指標進行算術運算與常規類型不同。只允許進行加法和減法運算，根據它們指向的數據類型的大小，加法和減法對指針的行為略有不同。

因為 ++ 比 * 有更高的優先級，最好使用括號通過增加表達式的易讀性來減少混淆。

# 指標和常量

指針可用於通過地址訪問變量，這種訪問可能包括修改指向的值。但是也可以聲明可以訪問指向值的指針來讀取它，但不能修改它。

const 指標的一個功能為函數參數，將指向非 const 參數的指標作為參數的函數，可以修改作為參數傳遞的值，而將指標作為 const 參數的函數則不能。

# 指標和字串

字符串是包含結尾的字符序列的數組。
可以透過指標直接訪問它們，每個元素都是類型 const char，作為文字，它們永遠不能被修改。

# 指向指標的指標

C++ 允許使用指向指標的指標。

  • c : 類型是 char** 、值是 8092 。
  • *c : 類型是 char* 、值是 7230 。
  • **c : 類型是 char 、值是 'z' 。

# 無效指標和空指標

在 C++ 中，void 表示缺少類型，因此 void 指標是指向沒有類型的值的指針，它的一種用途是將通用參數傳遞給函數。

指標可以指向任何地址，但必須是有效元素的地址，以下語句可以編譯成功，但執行時可能導致錯誤。

指標需要顯式任意地方，同時需避免無效地址。

不要將空指針與 void* 混淆！空指標是表示它指向無，而 void* 是一種沒有特定類型的指標。

# 函數指標

C++ 允許使用指向函數的指針進行操作，典型用途是將一個函數作為參數傳遞給另一個函數。

之前使用的經驗就是呼叫 DLL，抓個指標來賦型即可使用。
