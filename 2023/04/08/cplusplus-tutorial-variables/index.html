C++ 學習筆記 - 變量和類型

發表於2023-04-08|更新於2024-11-14

# 變量和類型介紹

  • 變量：可以將值存儲到內存中，方便進行操作和計算。
  • 類型：變量可以有不同的類型，例如整數、浮點數、布林值等，C++ 需要在變量名稱之前聲明其類型。

# 標識符 (Identifiers)

# 常見關鍵字

using, namespace, void, return
int, float, double, char, bool, signed, unsigned, long, short ,const , true, false, sizeof
if, else, for, do, while, switch, case, continue, break
class, struct, private, public, this, new, delete

# 自定義標識符

  1. 由字母、數字和下劃線組成
  2. 第 1 個符號不能是數字
  3. 長度：不同編譯器規定不同
  4. 區分大小寫（大小寫敏感）

非常重要 ：C++ 語言是一種 “區分大小寫” 的語言。例如 : RESULT 、 result 、 Result 這是三個不同的標識符，用於標識三個不同的變量。

# 變量 (Variable)

# 宣告

要求配置記憶體空間 註冊新名稱。

# 指派

將右值指派給左值。

# 取值

只要變數名稱不是出現在 = 左側，便是取得變數所代表的記憶體中的值。

# 隱式類型轉換

隱式類型轉換 (Implicit conversions) 也稱為自動類型轉換，編譯器自行完成，無需用戶的任何外部觸發。

  • 通常在表達式中存在多種類型時發生，會類型轉換以避免丟失數據
  • 數據類型會升級為更大的數據類型
  • 隱式轉換可能會丟失信息，並且可能發生溢出

# 顯式類型轉換

顯式類型轉換 (Casting) 也稱為類型轉換，用戶可以進行類型轉換，使其成為特定的數據類型。

  • 這樣做可利於應用結構或類型的某些特性
  • 助於計算包含不同數據類型變量的表達式

# 整數類 (Integer types)

C++ 語言的整數型態可概略分成 short 、 int 、 long 、 long long ，差異為佔用的記憶體空間不同，例如 short 是 16 位元，可儲存 -32768 到 32767 之間的整數值。 long 被稱為長整數，則至少佔有 4 個 byte 的記憶體空間，也就是 32 位元，可儲存 -2147483648 到 2147483647 之間的整數。

# short

描述 : short 是 16 位元的整數型別。
short 變數可儲存 16 位元的整數值，範圍由 -32768 到 32767 (最小值 -2^15，最大值 (2^15) – 1)。

# long

描述 : Long 是一個延伸儲存空間的變數型別，它可以儲存 32bits (4 bytes) 大小的整數，從 -2,147,483,648 到 2,147,483,647。

# long long

描述 : Long long 是一個延伸儲存空間的變數型別，它可以儲存 64bits (8 bytes) 大小的整數，從 –9,223,372,036,854,775,808 到 9,223,372,036,854,775,807。

# int

描述：整數型別是儲存一個數字的主要資料型別。
int 很特別，通常使用整數指的就是它，int 的範圍大小跟機器硬體的整數大小相同，並非固定為 32 位元或 64 位元，例如使用 32 位元的機器， int 與 long 的範圍大小會相同，所以實際上運用視所用機器及編譯器可否支援。

# unsigned int

描述：無號整數和有號整數一樣，都儲存在 4 byte (32 bits) 的空間，因為不能儲存負數，因此儲存正整數的範圍擴大為 0 到 4,294,967,295 (2^32 – 1)。

# 浮點數類 (Floating-point types)

類似於整數常數，使用浮點數常數亦可增加程式碼的可讀性，在編譯程式時，編譯器會自動將浮點數常數轉換成可用於運算的數值。浮點數常數可以被表示為科學記號 E 或者 e，即指數表示法。

# float

描述 : float 浮點數是一種含有小數點的資料型別。因為浮點數比整數有較佳的解析度，它可用來近似類比且連續的數值，浮點數範圍最大可至 3.4028235E+38，而最小可至 -3.4028235E+38，它們以 32 位元資訊儲存。

浮點數的數值並不是精確的，在比較時可能產生奇怪的結果；例如 6.0/3.0 可能不等於 2.0，而此時你應該檢查兩個比較數值的差的絕對值是否小於很小的數值。

在計算效能上，浮點數運算較整數運算慢，所以應該避免經常使用它。程式設計師通常會用一些方法將浮點數計算轉換整數運算來加快速度。

如果要用浮點數運算，你必須加上小數點，否則會被當成 int 運算。詳細資料可參考 浮點數 頁面。

# double

描述 : double 稱為雙精度浮點數，double 型別占用 8 個位元組 (64 bit) 記憶體，其可以表示的最大範圍是 1.79769E+308 到 -1.79769E+308，詳細的資料可以參考 二進位浮點數算術標準 IEEE754。

補充資料 : C/C++ 中各種不同型別的後綴

# 字元類 (Character types)

字元型態的關鍵字 char ，可以稍後利用指派給初值，凡是用單引號圍起來的單一字元都屬於字元型態，這樣就把字元  ‘A’  指派到變數 mychar 裡了。

# ASCII 碼

字元在電腦中是以一個八位元的整數來儲存 (即 1 Byte)，而這個符號與數字的對應關係我們稱為 ASCII 碼 (American Standard Code for Information Interchange)，也就是說其實字元也是一個數字，因此也可以拿來做加減乘除等四則運算。

# 字元陣列

字串就是一段文字，我們可以用一對雙引號  "文字"  把該段文字夾起來。由於 C++ 語言中並沒有字串的變數型態，而是用字元的陣列來儲存一個字串，字串型態 string 也就是連續字元的集合， \0 這是字串的結束字元， \0 不會被計算在字串的長度內，但每個字串都會自動附加上 \0 ，作為字串結束的標記。

# 字串和字元陣列

具有 null 結尾的字符序列是 C 語言中用來表示字符串的典型類型，即使在 C++ 中標準庫為字符串定義了特定類型字符串 (string)，C 字符串仍然很常見，並且大多數需要字符串的函數都被重載以支持兩者。

可以將字符序列轉換為字符串，string 的成員函數 c_str 可以將字符串轉換為字符序列。
