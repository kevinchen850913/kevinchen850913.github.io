C++ 學習筆記 - 基本程式架構

發表於2023-04-07|更新於2024-11-14

# C++ 基本要素

# Hello World

通常第一個程式是名為 “Hello World” 的程式，它只是將 “Hello World” 顯示到電腦螢幕上，雖然簡單但包含了 C++ 的基本組件。

# 標識符

C++ 學習筆記 - 變量和類型
C++ 學習筆記 - 常量 (Constants)

  • 標識符：程式中變數、類型、函式和標號的名稱。
  • 關鍵字：系統已有的標識符。
  • 自訂義標識符：自己給變數、函數等起的名字。

# 運算子

C++ 學習筆記 - 運算符 (Operators)

  • 算數運算符號 : +, - , * , /
  • 關係運算子 : > , <

# 陳述式

C++ 學習筆記 - 流程控制
C++ 程式由一系列語句組成，語句是可以產生某種效果的表達式。在 main 函數主體部分中，可以使用各種語句來進行變數聲明、運算、流程控制、函數調用等操作。每一段的程式碼都需要用分號 ; ，它代表一個表達完整的命令。

# 輸入 、 輸出

C++ 學習筆記 - 基本輸入輸出
在 C++ 中，使用 cin 和 cout 以在終端機獲取和輸出資料。

# 函數 function

C++ 學習筆記 - 特殊函數
能完成一定功能的有名的程式序列，是組成 C++ 程序的基本單位。

# 命名空間標準

C++ 學習筆記 - 名稱可見性
命名空間是 C++ 用來區分不同名稱之間的作用域的一種機制，使用元素時可以限定其命名空間，以避免命名衝突。

# 預處理器指令

以 # 開頭的行是由預處理器解釋的指令，它們是在程序本身的編譯開始之前解釋的特殊行，其中 #include 指令是最常見的一種，用於引入標準或自定義的標頭檔。

# 工程師的好習慣

# 註解

程式員可以在程式碼中插入註解以提高可讀性，其對程式行為沒有影響，例如範例中的 // my first program in C++ 就沒有實質行為。

C++ 支持兩種註解的方式

  • 行註解 : // 的位置到同一行末尾的所有內容。
  • 區塊註解 : /* 字符和字符第一次出現之間的所有內容 */ 。

# 縮排

為了讓程序更易讀，C++ 中使用不同的行結構和適當的縮進，但沒有關於縮進或如何在不同行中拆分指令的嚴格規則。下方使用函數 main 的寫法為例。

# 編碼風格

wiki 編碼風格
集合種習慣並統一規則，就會成為編碼風格，要成為好的工程師一定要有編碼風格，比起在意採用何種風格，更重要的是在團隊中統一編碼風格，使程式碼更有一致性，若是初學者建議 Visual Studio 裝起來，模仿自動排版的格式即可。

# 建議學習順序

  • 變量、數據類型和數值運算符
  • 基本輸入與輸出
  • 邏輯 (if 語句、switch 語句)

# 常見編譯錯誤

  • 忘記加分號：解決方法在錯誤出現處的行末尾加上 ; 。
  • 括號沒括好：解決方法檢查自己縮排的狀況，並找出遺漏的 {或} 並補上。
  • 忘記宣告：解決方法檢查使用前已被宣告，若無則就宣告它。
