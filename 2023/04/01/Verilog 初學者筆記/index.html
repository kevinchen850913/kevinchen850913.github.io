# 關於寫法

# Notgate

! 與 ~ 所做的事是不一樣的，只有在 1bit 時動作是一樣的。

  • ! 是邏輯的反向， !0x00 = true
  • ~ 是位元的反向， ~0x00 = 0xFF

除此之外還會常用到其他位元運算

  • NOR
  • XNOR
  • XOR

判斷 a 相等 b 有更快的方法， a ^ b = 0 比 a == b 更快，異或運算的效率更高。

# reg 不是變數

wire reg 不能像 C 語言中一個變數用到底，這樣寫會有問題。

需手動處理資料合併問題。

# always 用法

對於硬件綜合有兩種相關的 always 塊

  • 組合邏輯： always@(*)
  • 時序邏輯： always@(posedge clk)

時鐘控制的總是塊創建一個組合邏輯塊，就像組合總是塊一樣，而且還在組合邏輯塊的輸出處創建一組觸發器。不是立即可見邏輯塊的輸出，而是僅在下一個（posedge clk）之後立即可見輸出。

# 阻塞與非阻塞

Verilog 中有三種類型的賦值

  • 連續賦值（assign x = y;）。只能在不在過程中使用。
  • 程序塊分配：( x = y; )。只能在程序內部使用。
  • 程序非阻塞賦值：( x <= y; )。只能在程序內部使用。
    在合的 always 塊中，使用塊分配。在時鐘控制的 always 塊中，使用非阻塞分配。完全理解為什麼對硬件設計不是特別有用，並且需要很好地理解 Verilog 模擬器如何跟踪事件。不遵循此規則會導致極難發現既不確定又在模擬和合成硬件之間存在差異的錯誤。

在組合塊中，使用阻塞賦值 (=)，在時鐘塊中，使用非阻塞分配 (<=)。

# 路過不做事

上面這段程式，什麼事也沒做，就只是單純的把數值，從上一個模組往下一個模組丟，在硬體中傳值就是拉一條線過去，考慮資料的依賴性，往往會經過許多 過水 的模組，直到最後一個模組才開始做事。

  • 資料流一開始就要想清楚，不然很難修改。
  • 不是最後一關就不要動，手腳乾淨點。

# 不延遲狀態機

一般狀態轉移時會消耗一 clk，但若必須同時輸出結果，此時則必須搭配組合電路寫法，用組合電路轉移狀態，時序電路輸出，這樣才能及時的同步輸出。

https://hdlbits.01xz.net/wiki/Fsm_hdlc

  • data 是因時序平移，所以必然會延遲。
  • s_disc 必須同時使用 data + in 才能準時完成。

個人理解：延遲離散化，要撐到輸出的最後一刻再固定數值。
