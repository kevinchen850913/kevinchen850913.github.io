FAE 工程師在做什麼

發表於2023-03-12|更新於2024-11-14

# FAE 是什麼意思？

FAE = Field Application Engineer
FAE = Failure Analysis Engineer
Fucking Ass Engineer, just a kidding
英文的縮寫就是個玩笑，文章中指的是現場應用工程師

# FAE 的職場定位？

FAE（Field Application Engineer）是產業中的一個職位，負責協助客戶導入自家產品，提供售前、售後的顧問諮詢服務。這時候就會有人好奇，那薪資水平跟技術難度的有無正相關，一般來說技術難度高的工作單位時薪會比較好，不過薪水主要還是取決於產業發展狀況與加班時數，而技術難度並非是關鍵因素。

職位與技術難度鍊 (越左邊越困難)
豬屎屋 RD > 豬屎屋 FAE > 整合 RD > 整合 FAE > 設備 RD > 設備 FAE
豬屎屋 FAE : 通常是資深 RD 下來養老。
整合 FAE : 嘗試解決問題 + 下跪。
設備 FAE : 專門負責去客戶端下跪。

FAE 的技術能力無論再強大，都還是要依靠別人的幫助，才能夠存活下去，所以不管公司內部還是客戶的關係都要搞好，但台廠大多都把 FAE 當做第一線的炮灰在消耗，幫公司或是產品揹黑鍋，反正人走了就再來抓交替，導致新人容易被捅刀，客戶、PM、RD、部門內的同事，自信心不足的人很容易受到挫折，不過你可以換個角度想，你本來就是來跪的，只要跪的夠低別人就桶不了你，而且累積經驗後就會越躺越舒服。

# FAE 的工作內容？

FAE 的工作方式可以分為偏技術和偏業務兩個方向，類似於顧問與業務的綜合體，需要在產品售前、售後為顧客提供顧問服務，或是當技術人員直接面對客戶解決技術相關疑難問題，並且協助客戶導入產品。

技術 FAE : 幫客戶在現場解決問題。
業務 FAE : 嘗試幫客戶凹研發單位解決問題。

基本上能讓客戶滿意的就是好 FAE，比起傳統的工程師，FAE 更有即興發揮的空間，一百位 FAE、可能有一百種方法，這些分類應該能解答，為什麼會有人說 FAE 適合新鮮人，又有另一派說 FAE 最好要有資深 RD 的經驗。

# FAE 需要具備的能力？

為了勝任上述工作內容，FAE 工程師需要的多元能力，包含專業技術、了解市場產品、溝通表達力、和抗壓能力，綜覽如下：

  1. 專業技術：為客戶提供技術支援，快速分辨客戶遇到的問題，是來自哪個系統，並提出解決方案。
  2. 了解市場產品：一個成熟的 FAE，一定對自家產品、客戶系統、甚至市場上競爭對手的產品瞭若指掌，因為客戶可能使用了十幾家不同的產品，FAE 需要掌握這些知識，才能順利推薦合適的產品，同時也避免被問倒。
  3. 溝通與表達：介紹自家解決方案、或是協助客戶使用產品，都需要讓客戶願意聽、並且聽懂你的說明，將服務化為實際的訂單；對內，需要將客戶需求傳達到 RD 部門確保客製功能或零件符合客戶期待。
  4. 抗壓性與應變力：對需要與客戶朝夕相處的 FAE 而言，面對各種意想不到的要求，需要抗壓性與靈機應變的能力，才能保持機動性、維繫與客戶的關係。

如果你擁有上述的能力，我建議你可以創業當老闆。你非常優秀，不應該讓自己成為別人的奴才。實際上只要擁有這些能力中的任何一項，你都可以嘗試投履歷，在面試官眼中，你就像是一個產品，而他們是客戶，如果你成功進入公司，你就一定能夠勝任 FAE 的工作，所以放心去投不用想太多。

# FAE 有前途嗎？

就技術層面，晶片設計原廠 FAE 可以獲得比台廠 RD 更多的技術資料，所以有機會在技術上取得突破，但如果是整合商 FAE 或是設備商 FAE 就會比較少直接接觸到專業技術。

在公司內部，RD 往往都是負責一小塊特定的領域，但作為 FAE 會跟著客戶案件跑，所以在跨職位、跨領域都會比較有優勢，問題碰多了累積足夠的經驗，在實戰上也能磨出自己的見解，後續再去補上理論知識即可成功轉換跑道。

FAE 能累積更多的人脈，有時候成功的關鍵不是個人的能力與努力，而是運氣主導一切，這時候有沒有認識 "貴人"，就決定了你能不能翻身，拓展更大的圈子，就有更高的機率能夠把握機會扭轉命運。

# FAE 適不適合我？

  • 職場新鮮人適合 ，因為可以快速了解職場生態。
  • 想創業的人適合，因為有市場、人脈、客源等優勢。
  • 想提昇技術的人，只適合晶片原廠 FAE，其他的不適合。
  • 脾氣不好的人不適合，因為總會遇到要下跪的時候。

這篇文章為我個人經驗整理，從各個面向去探討 FAE 的工作內容，並補足一些網路上比較不夠完整的答案，相信可以幫助讀者找到適合自己的方向，早日脫離求職焦慮。
