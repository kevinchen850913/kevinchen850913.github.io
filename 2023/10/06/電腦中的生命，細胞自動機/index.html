電腦中的生命，細胞自動機

發表於2023-10-06|更新於2024-11-12

# 生命遊戲介紹

生命遊戲不是真正的遊戲。沒有勝利或失敗，也沒有在精神上和精神上摧毀你的對手。生命是一個細胞自動機，一個生活在網格上的細胞系統，它們根據統治其世界的規則生活、死亡和進化，它包括一個二維矩形世界，這個世界中的每個方格居住著一個活著的或死了的細胞。一個細胞在下一個時刻生死取決於相鄰八個方格中活著的或死了的細胞的數量，比賽按回合進行。在每一輪中，每個細胞都會查看其 8 個最接近的鄰居，並計算當前存活的細胞數量。

生命遊戲中，對於任意細胞，規則如下：

  • 當前細胞為存活狀態時，當周圍的存活細胞低於 2 個時，該細胞變成死亡狀態。
  • 當前細胞為存活狀態時，當周圍有 2 個或 3 個存活細胞時，該細胞保持原樣。
  • 當前細胞為存活狀態時，當周圍有超過 3 個存活細胞時，該細胞變成死亡狀態。
  • 當前細胞為死亡狀態時，當周圍有 3 個存活細胞時，該細胞變成存活狀態。

# 儲存板狀態

生活遊戲是一個二維網格，其中每個單元格都為 ALIVE (1) 或 DEAD (0)。

建立一個名為 init_board 的函數，函數將接受 2 個參數，板的寬度和高度。

建立一個名為 random_board 的函數，它將返回一個棋盤狀態，其中每個單元格都已隨機初始化為 ALIVE 或 DEAD ，這種隨機模式被稱為湯，它們是我們開始產生有趣輸出的最快方式。

# 將板列印到終端

您已經可以使用您語言的標準 print 功能將電路板列印到終端，在這個步驟中，我們將編寫一個 show_board 函數，使終端出現你喜歡的外觀。

# 計算下一個狀態

當開始計算下一行的新狀態時，將使用目前行的新活動值，應該用來計算下一行的新狀態的值，如果直接在原有的板上修改，程式不會拋出任何錯誤，但會給出一些非常奇怪的結果。

# 計算存活鄰居的數量

單元格 (x, y) 的鄰居是在任意方向上與其兩側相隔一個間隔的單元格，把這些還活著的鄰居的數量加起來，記得處理邊緣的儲存格，如果發生溢位，則可能會遇到問題。

# 永遠運行生命遊戲

現在是我們創造永生的時候了。在無限循環的每次迭代之後，我們需要使用我們的函數將新的棋盤狀態列印到終端，它看起來就像一個奇特的、自動更新的使用者介面。
