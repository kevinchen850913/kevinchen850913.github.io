C++ 學習筆記 - 特殊成員函數

發表於2023-07-07|更新於2024-11-12
|閱讀量:

# 特殊成員函數介紹

特殊成員函式是類別成員函式，在某些情況下，編譯器會自動為您產生。些函式是預設建構函式 (Default constructor)、 解構函式 (Destructor)、複製建構函式 (Copy constructor)、複製指派運算子 (Copy assignment)、移動建構函式 (Move constructor) 和移動指派運算子 (Move assignment)。

Member function typical form for class  C :
Default constructor C::C();
Destructor C::~C();
Copy constructor C::C (const C&);
Copy assignment C& operator= (const C&);
Move constructor C::C (C&&);
Move assignment C& operator= (C&&);

Rule of three (C++ programming) : 基於安全考量，最好顯式定義 析構函數、複製構造函數、複製賦值運算符，當然如果可以就全部顯式定義 (Rule of five)。

# Default constructor

默認建構子 (Default Constructor) 是在聲明類的對象時，如果沒有使用任何參數進行初始化，則會被調用的構造函數。如果一個類的定義沒有聲明任何構造函數，編譯器將隱式地定義一個默認構造函數，默認構造函數沒有參數，並且可以執行必要的初始化操作。

一旦類中聲明了帶有任意數量參數的構造函數，編譯器將不再提供隱式的默認構造函數。這意味著如果需要在不帶參數的情況下構造該類的對象，就需要顯式地聲明並實現一個帶有空參數的默認構造函數。這樣可以確保該類的對象可以在沒有參數的情況下被正確構造和初始化。

需要注意的是，默認建構子可以被重載和定義，以便進行更多的初始化操作。根據需要，可以提供具有不同參數的多個默認構造函數。這樣可以根據不同的情況，使用不同的初始化方式來創建對象。

# Destructor

析構函數 (Destructor) 是一個特殊的成員函數，與構造函數相反。它在對象的生命週期結束時被自動調用，用於進行必要的清理工作，例如釋放動態內存、關閉文件或斷開連接。析構函數沒有參數，也不返回值。它的目的是確保在對象不再需要時進行適當的清理操作，以防止資源泄漏和不正確的狀態。

# Copy constructors and copy assignment

複製構造函數 (Copy Constructor) 和複製賦值運算符 (Copy Assignment) 用於在同一類型的對象之間進行複製操作。

複製構造函數的目的是創建一個新的對象，並將源對象的值複製到新對象中。默認情況下，編譯器提供隱式複製構造函數，執行淺複製，即僅複製對象的成員，而不複製動態分配的資源。

如果類包含指針成員或需要深層複製的其他特殊情況，則需要自定義複製構造函數以執行深複製操作，確保每個對象都有自己的資源副本。

複製賦值運算符用於在已有的對象上將另一個對象的值複製到它上面。它的目的是使一個對象等於另一個對象，並且在賦值操作後，兩個對象將具有相同的值。默認情況下，編譯器提供隱式複製賦值運算符，執行淺複製。

同樣地，如果需要自定義深複製的操作，則需要定義自定義的複製賦值運算符。這將確保在賦值操作中進行深複製，而不僅僅是複製成員值。

# Move constructor and assignment

移動構造函數 (Move Constructor) 和移動賦值運算符 (Move Assignment) 與復制類似，它們用於將值從一個對象轉移給另一個對象。然而，移動的特點在於實際上將內容從源對象轉移到目標對象，源對象失去了該內容，而目標對象接管了該內容。

移動操作通常在未命名的臨時對象上發生，這些臨時對象是函數返回值或類型轉換的結果。編譯器對於這些情況已經進行了優化，稱為 “返回值優化”，在某些情況下，移動構造函數實際上可能不會被調用。

移動構造函數的目的是高效地轉移資源，例如動態分配的內存或其他資源，從一個對象到另一個對象，而無需進行深層複製操作。移動構造函數通常會將源對象的內部指針或資源所有權轉移到目標對象，以避免不必要的資源複製或釋放。

同樣地，移動賦值運算符用於將源對象的值移動到目標對象上。這通常在對象已存在且需要更新其值時發生。移動賦值運算符的目的是高效地轉移資源，並避免不必要的資源複製或釋放。

需要注意的是，如果類定義了自定義的復制構造函數或復制賦值運算符，則可能需要定義相應的移動構造函數和移動賦值運算符，以確保對象的內容正確且高效地移動。

# Default constructor vs Copy constructor

  • Default constructor，沒有參數 C() 的建構子。
  • Copy constructor，有參數 C(const C&) 的建構子。
    補充一個細節就是 Default constructor 在使用上不可以加上括號，要寫成 Example e; ，而不是 Example e(); 。

# Copy constructor vs Move constructor

  • Copy constructor，複製貼上，參數是 const C& (唯獨)。
  • Move constructor，剪下貼上，參數是 C&& (可刪除)。
    Copy 與 Move 的差異就是作為參數的實體是否會被消滅。

# Constructor vs Assignment

  • Constructor，從無產生出實體。
  • Assignment，實體存在左值，需左值承接。
    Constructor 的用法為 Example a, b; ，而 Assignment 會比較類似 new ，關鍵字是 operator= ，用法為 a = b; 。

提供一個不精確的例子，建構類別就像學習知識。

  • Default constructor : 沒讀書，無師自通。
  • Copy constructor : 自己讀書。
  • Copy assignment : 同儕讀書後來教你。
  • Move constructor : 自己讀書後筆記遺失。
  • Move assignment : 同儕讀書後來教你，然後筆記遺失。
