C++ 學習筆記 - 友誼與繼承

發表於2023-07-14|更新於2024-11-14

# Friend Functions - 友誼函數

原則上，類的私有成員和受保護成員不能從聲明它們的同一類的外部訪問。不過，這條規則不適用於 friend 。

如果非成員函數被聲明為該類的朋友，則它可以訪問該類的私有成員和受保護成員。這是通過在類中包含此外部函數的聲明並在其前面加上關鍵字 friend 來完成。

友誼關鍵字應該謹慎使用。如果一個擁有 private 或者 protected 成員的類別，宣告過多的友誼函數，可能會降低封裝性的價值，也可能對整個設計框架產生影響。

# Friend classes - 友誼類別

與友誼函數類似，友誼類是其成員可以訪問另一個類的私有或受保護成員的類。除非指定，否則友誼永遠不會對應示例中，Rectangle 被視為好友類 Square，但 Square 不被 視為好友 Rectangle。友誼的另一個屬性是它們不具有傳遞性，朋友的朋友不被視為朋友。

# 類之間的繼承

C++ 中的類可以透過繼承來擴展，創建保留基類特徵的新類。繼承涉及基類和派生類的關係，派生類可以繼承基類的成員並添加自己的成員。

舉例來說，我們可以想像一系列描述多邊形的類，包括矩形和三角形。這兩種多邊形有一些共同的屬性，例如面積的計算方式可能都需要高度和寬度（或底邊）。

我們可以使用一個基類 Polygon 來表示這些共同特徵，並從這個基類派生出兩個類：Rectangle（矩形）和 Triangle（三角形）

同類別成員 是的 是的 是的
派生類成員 是的 是的 不
不是成員 是的 不 不

# 從基類繼承了什麼

公共派生類繼承對基類 不包含 :

  • 它的構造函數和析構函數
  • 其賦值運算符成員 (operator=)
  • 它的朋友
  • 其私人成員

儘管對基類的構造函數和析構函數的訪問不是繼承的，但派生類的構造函數和析構函數會自動調用它們。

# 多重繼承

一個類可以從多個類繼承，只需在類的基類列表中（即冒號後面）指定多個基類（用逗號分隔）即可。
