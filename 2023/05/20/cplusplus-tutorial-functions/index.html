# 沒有類型的函數

如果函數不需要返回值，此時要使用的類型是 void ，這是一種表示值缺失的特殊類型。

  • 表示值缺失時使用 void
  • C++ 中函數一定要使用 ()
  • 參數列表中 (void) 等同於 ()

範例：只打印消息的函數。

括號是將函數與其他類型的聲明或語句區分開來的東西。

  • 函數 : printmessage ();
  • 不是函數 : printmessage;

# main 函數

int main()
main 是所有 C++ 程序中的特殊函數，程式的執行從這裡開始。在 main 函數中，可以定義回傳值、輸入參數等。

# main 的參數

在 C++ 中，fun () 和 fun (void) 是等價的，但在 C 中，int main () 可以通過任意數量的參數調用，而 int main (void) 只能無參調用。雖然大部分時候二者沒什麼區別，但是在 C 中，推薦使用 int main (void) 寫法。

# main 的返回值

在大多數範例中 main 沒有返回任何值。
若執行 main 正常結束而沒有遇到 return 語句，則編譯器假定函數以隱式 return 語句結束。

main 是特例，其他函數都應包含 return 語句作為結束。

# 按值和按址傳遞的參數

# 按值傳遞

當一個函數被調用並傳遞了兩個值（5 和 3）作為參數時，這些值分別被複製到函數定義中設置的變量（x 和 y）中。

這些變量在函數內部使用，但對它們的任何修改都不會影響函數外部的原始變量（x 和 y）的值。

這是因為在函數調用時，只是將這些值的副本傳遞給了函數，而不是將原始變量本身傳遞給函數。

# 按址傳遞

在某些情況下，我們希望從函數內部直接訪問並修改外部變量的值。為實現這一目的，我們可以使用引用參數而不是值參數。引用允許我們在函數中使用外部變量的別名，從而可以直接修改該變量的值。

# 效率考慮和 const 引用

按值傳遞比較慢。

按址傳遞比較快，但不想被修改。

按址傳遞 + 唯讀，兩個願望一次滿足，但不一定比較快。

const 引用提供了類似於按值傳遞參數的功能，但提高了大型參數的效率。這就是為什麼它們在 C++ 中非常流行用於復合類型的參數。但請注意，對於大多數基本類型，效率沒有明顯差異，在某些情況下，const 引用甚至可能效率更低！

# 內聯函數

調用函數通常會導致一定的開銷，有時在調用函數的地方插入函數代碼可能更有效，而不是執行過程正式調用一個函數。

為了實現這種需求，可以在函數聲明之前加上 inline 關鍵字，以提示編譯器優先進行內聯擴展而不是使用常規的函數調用機制。

這並不會改變函數的行為，而僅僅是向編譯器建議，在每個調用點插入函數體生成的代碼，以避免函數調用的開銷。

請注意，大多數編譯器會自動優化代碼，生成內聯函數，即使沒有標記符 inline 。

# 參數中的默認值

函數有可選參數，在調用中不需要參數，例如一個有三個參數的函數可以只用兩個調用。為此，該函數應包括其最後一個參數的默認值，該函數在使用較少參數調用時使用該默認值。

第二次調用中將兩個參數傳遞給該函數。
因此忽略 b 的默認值，並將傳遞的值作為參數。
