如何控制 PCIE 裝置

發表於2023-08-15|更新於2024-01-20
|閱讀量:

# 入門須知

本教學旨在讓讀者成功進行 PCIe 的讀寫操作，因此將避免過多的理論解釋。我們的目標非常明確：讓讀者能夠實際操作，並順利完成 PCIe 的讀寫任務。

位置 = 匯流排 (Bus) + 裝置 (Device) + 函數 (Function)
本教學中：匯流排 = 2 裝置 = 0 函數 = 0

識別碼 = Vendor ID + Device ID
本教學中 : Vendor ID = 0x10EE Device ID = 0x7011

PCIE 會有固定的 CONFIG REGUSTERS，其中會提供 BAR 空間位置，然後大部分的功能都會在 BAR 中運行，現在你已經知道最基本的知識，可以開始實際操作了。

# RW Everything 驗證

RW 網站，這個工具常常會用到。

選擇正確的匯流排 (Bus) + 裝置 (Device) + 函數 (Function)，就可以找到裝置，其中也有包含開頭提到的所有資訊。

依照上圖說明，BAR1 = 0xF7D40000，然後我又在裝置的特定區段中使其回傳 0x12345678 ，在使用上要指定 BAR1 再加上 Offset (0xA4 我自己設計的回傳位置)，就可以看到如下方的資訊，代表裝置能正常運作。

# 實際測試狀況

簡單寫個程式持續讀寫，若回傳不是 0x12345678，則判斷異常，這樣可以確保每次都可以取得正確的資料，然後就丟著讓他自己跑即可。

教學中的驅動程式是使用 WDK 開發，若用其他方式開發，下方提供程式碼不一定能正常使用，需自行留意。

# 測試程式碼範例

你只要改 CreateHandle 的參數就能用，然後測資判斷條件就因需求設計，不過最好盡量確保地址正確，不然電腦可能會藍屏或是直接重開機。
