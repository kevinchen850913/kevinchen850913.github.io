Hexo 架設教學與踩坑紀錄

發表於2023-08-01|更新於2024-08-30
|閱讀量:

# Node.js 安裝

https://nodejs.org/en/download/
Node.js 是一個 JavaScript 執行環境 (run-time environment)。原本 JavaScript 檔案只能在瀏覽器上被執行，但是當我們在電腦上安裝了 Node.js 之後，就可以執行副檔名為 .js 的 JavaScript 的檔案。

在 cmd 輸入 node -v 來檢查是否工作正常，有返回就行。

# Git 安裝

我每台電腦都有裝，所以不需要處理這一步，需要的人可以去 Git 官網下載。

# Hexo 安裝

hexo 官方文件

  1. 在合適的位置新建一個文件夾 ，用於存放你的 hexo 博客文件位置。
  2. 進入文件夾，按住 shift 的同時點擊右鍵 ，在右鍵菜單中點擊 在此處打開 powershell 窗口
  3. 輸入命令 npm i -g hexo-cli 進行安裝

# Hexo 初始化

輸入 hexo init 初始化以及 npm install 安裝必要組件

# Npm 版本太舊

nodejs 的版本更新後，就會導致 npm 和 nodejs 不匹配，不過因為我電腦的環境單純，所以直接統一更新到最新版本即可，不會有問題。

# Hexo 安裝失敗

我第二次架設的時候，多次安裝失敗，原因是因為資料夾不是空的，我本來是先 clone github 的 repository 下來操作，不過在 clone 下來的資料夾初始化 Hexo 會一直告訴我資料夾不是空的，啟動失敗，所以後來決定還是先從空的資料夾來初始化 Hexo。

正確流程：空資料夾 => 安裝 Hexo => 初始化 Hexo => clone github

# Hexo 初始化失敗

hexo 錯誤資訊：因為這個系統上已停用指令碼執行，所以無法載入

由 powershell 改用 Git Bash 就成功，不知道電腦又在搞什麼 ，Git Bash 是 Windows 版的 Git 模擬器，可使用 Linux 指令操作 Window 的檔案系統。
powershell => Git Bash

# Hexo 基本指令

  • 清空 : hexo clean
  • 建立 : hexo g
  • 部屬 : hexo s
  • 建立新頁 : hexo new “在此輸入檔案名稱”
  • 更新至 github : hexo d

# Hexo 建立失敗

在 themes 資料夾中確認主題資料夾存在，且名稱完全一致，主題的安裝方法請參考各主題教學，補充主體資料夾中含有 .git 意味著該資料夾已被 git 版本控管，此時就不會再被更外層的庫給列管，所以若是無自動更新需求，可以將主體資料夾中的 .git 刪除，使其中的檔案可以被統一管理。

# 本地端測試

hexo clean , hexo g , hexo s 三連，瀏覽器輸入 localhost:4000 ，點開默認的博文 hello world ，查看網頁是否渲染正常輕輕鬆鬆解決。

會看到顯示的畫面，running at http://localhost:4000/

# Hexo 草稿功能

Hexo 另外提供 draft 機制，它的原理是新文章將建立在 source/_drafts 目錄下，hexo generate 不會編譯 source/_drafts 目錄下的文章，但 Hexo 的 Hexo server 另外提供 --draft 參數，這讓我們只要搭配 hexo-browsersync plugins ，就可以達到一邊編輯 markdown 文章，一邊使用瀏覽器預覽的目的。
