自製剪貼簿工具(Clipboard)

發表於2023-09-22|更新於2024-08-30
|閱讀量:

# 故事起因

我習慣使用 md 檔撰寫筆記，而文字外的檔案就由連結來管理，但時間一久資料越來越多，於是一直想把文字跟其他資料進行切割，讓筆記庫只有文字，其他檔案都該放在其他地方，但是跨工作區的資料連結，造成我不小的麻煩，如果使用預設的相對路徑 md 檔與資料無法分離，只能從絕對路徑下手，工人智慧處理路徑，操作起來十分繁瑣又浪費時間。

  • 想直接看成果，可以右邊目錄選 成品範例

md 檔外部路徑的範例

逐步冒出的問題

  • 無法使用工具原生的索引
  • 無法動態更新路路徑
  • 相對路徑還是無法切割檔案
  • 路徑越長，遇到越多特殊字元
  • 引用越來越麻煩，我越來越懶

總之該來正視這個問題了，所以才有了這篇文章。

# 解決思路

在尋找工具前，先想像有可能得到什麼，也許微軟有提供原生的快捷建，只是我不知道，也許有成熟的工具，可以直接使用，也許有半套的腳本工具，需要自己編寫一部份程式，所以我先整理了目標的優先順序。

目標需求 (重要 => 可有可無)

  1. 剪貼簿的讀寫與修改
  2. 剪貼簿的文字正規化
  3. 修改的快捷方法
  4. 複製路徑的快捷方法

補充複製路徑的方法與我的想法
壓住 shift 同時右鍵檔案，即會出現複製路徑的選項。這個操作方式有夠卡手，右鍵開選單都已經夠麻煩了，還要押壓住 shift 才會出現，然後複製出的文字會被加上雙引號，對使用者又增加一層困擾。

# 尋找方案

遇到問題就是先餵狗，沒想到竟然尋找不到現成方案，可能是會這麼用電腦的人是小眾中的小眾，而且有能力自己解決，看來還是只能自己寫，反正能有多難，先做個垃圾，反正之後還能改。

  • 右鍵選單擴充工具，太肥大，而且早期軟體支援性堪慮
  • 滑鼠自訂巨集，能自訂義的部分太少，還是要自己寫
  • 微軟救救我，你確定？改預設功能要修改機碼喔

# 絕對路徑對策

你或許有聽過，絕對路徑就是絕對不要使用的路徑，切勿直接使用絕對路徑，要特殊處理成類似相對路徑，還有檔案不能改名、不能移動，這些必要之惡要有心理準備，所以在制定檔案名稱時要想清楚，最好只使用英文跟數字，避免使用特殊符號，造成自己的困擾。

使用絕對路徑的老方法，自訂環境變數或是捷徑，解決彈性不足的問題，而這是我採用的方法是網路硬碟的綽號，檔案放在 NAS 個人覺得這樣是最直觀使用的方法，本機 => 連結網路磁碟機 => 自訂某個槽用來當路徑。

# 成品範例

確認物件功能撰寫讀寫測試，簡單測試 Clipboard 能否正常運作。

加上文字處理，為了避免特殊字元導致方法爆開，所以自己手動塞，到這部直接把應用程式綁在快捷鍵上，就能順利使用。

更進一步修改加上監察者，讓程式能在執行中就自動完成此工作，如果不在意效能，我也看到有人直接輪尋，使用方法，開啟程式後自動監看複製事件，若特定條件，則使連結自動修改再塞回剪貼簿，等我使用一段時間後再來分享心得。

實際應用有夠好用，想怎麼改就怎麼改，除了這支程式，我後來還去修改了機碼，直接把功能綁到滑鼠右鍵選單，文書作業處理路徑時加速 100%，不過真的覺得修改機碼有夠麻煩，微軟應該開放出來讓大家自己自訂，類似滑鼠的自訂義按鍵一樣。
